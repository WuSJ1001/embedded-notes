
# 第一步：安装Git插件
首先，确保你已经在电脑上安装了Git。如果还没有，可以去Git官网下载安装包，一路“下一步”即可。

**打开Obsidian → 设置 → 第三方插件 → 浏览**，在插件市场中搜索“Git”，找到它并安装。安装后记得**启用**这个插件。

**关键配置点**：在插件设置中找到 `Custom Git Binary Path`（自定义Git路径），这里需要填写你电脑上 `git.exe` 的完整路径。它通常位于Git安装目录下的 `bin` 文件夹里。

# 第二步：理解“仓库”概念
这里有个重要概念需要理解，对后续操作很有帮助：

- **Obsidian仓库(Vault）**：就是你存放所有笔记的那个文件夹。
- **Git仓库（Repository）**：Git用来管理版本历史的文件夹。

**我们要做的，就是把Obsidian仓库变成一个Git仓库**，这样Git就能跟踪你笔记的每一次变化了。

## 初始化本地仓库

在Obsidian中按 `Ctrl+P`（Windows/Linux）或 `Cmd+P`（Mac）打开命令面板，输入并执行：**“Git: Initialize a new repository”**。
这会在你的笔记文件夹中创建一个隐藏的 `.git` 文件夹，标志着Git开始接管你的笔记版本控制。

# 第三步：配置云端备份仓库（以Gitee为例）
本地有了版本控制还不够，我们需要一个云端“保险箱”。

1. **注册并登录Gitee**
2. **点击右上角“+” → 新建仓库**
3. **重要设置**：

- 仓库名称：建议用英文，如 `my-obsidian-notes`
- 勾选“私有”（**这点很重要**，除非你想公开所有笔记）
- 其他保持默认即可

创建完成后，**复制仓库的HTTPS链接**，稍后会用到。

## 设置远程仓库地址

回到Obsidian，打开命令面板，执行：**“Git: Edit remote”**。

- 远程名称填 `origin`（这是Git的惯例）
- URL粘贴刚才复制的Gitee仓库链接

## 获取访问“钥匙”——私人令牌
#SSH 

因为我们是私有仓库，不能直接用账号密码，需要用令牌访问：

1. 进入Gitee → 点击右上角头像 → 设置 → 安全设置 → 私人令牌
2. 点击“生成新令牌”
3. **至少勾选以下权限**：`projects`、`hook`
4. 生成后**立即复制保存**，这个令牌只显示一次！


# 第四步：开启同步之旅

现在一切就绪！在Obsidian命令面板执行：**“Git: Open source control view”**，你会看到Git操作面板。

# 高级技巧：自动同步

如果觉得手动同步麻烦，可以在Git插件设置中开启**自动同步**：

- 定时自动推送/拉取
- 文件保存后自动提交
- 启动Obsidian时自动同步
