在芯片设计（尤其是微控制器 MCU 或系统级芯片 SoC）中，存储单元根据其**速度、用途、是否掉电丢失数据**，可以分为多个层次。

我们可以按照从“距离 CPU 最近”到“距离 CPU 最远”的顺序来梳理：

### 1. 寄存器 (Registers) —— 速度最快
寄存器直接集成在 CPU 核心内部，是 CPU 进行运算时直接操作的对象。
*   **用途**：存放当前正在执行的指令、参与运算的操作数、中间结果以及状态信息（如之前提到的 **PC 程序计数器**）。
*   **特点**：速度极快（与 CPU 同频），数量非常有限（通常只有几十个），容量极小。

### 2. 高速缓存 (Cache) —— 性能桥梁
在高性能芯片（如手机、电脑处理器）中，CPU 速度远快于内存（RAM），为了不让 CPU 等待内存，会加入 Cache。
*   **类型**：通常分为一级缓存（L1）、二级缓存（L2）、三级缓存（L3）。
*   **原理**：利用“局部性原理”，预先从内存中加载 CPU 可能用到的数据。
*   **介质**：通常使用 **SRAM**（静态随机存储器）。

### 3. 随机存取存储器 (RAM) —— 工作台
RAM 是程序运行时的主要“操作间”，掉电后数据会丢失（易失性）。
*   **SRAM (Static RAM)**：
    *   **特点**：速度快，功耗低，但电路结构复杂，占用芯片面积大，所以容量通常较小（KB 到几 MB）。
    *   **应用**：嵌入式芯片（如蓝牙芯片、STM32）内部主要的运行内存。
*   **DRAM (Dynamic RAM)**：
    *   **特点**：集成度高，相同面积下容量大，但速度比 SRAM 慢，且需要不断刷新电流来维持数据。
    *   **应用**：电脑内存条、手机运行内存（LPDDR）。*注：大多数小型芯片内部不集成大容量 DRAM，而是通过总线外接。*

### 4. 只读存储器 (ROM) 系列 —— 基础固化
用于存放那些“永远不需要改动”或者“只有特殊情况才改动”的数据。
*   **Mask ROM (掩模 ROM)**：在芯片制造时就把数据刻死，完全不可更改。通常用于存放最底层的引导程序（Bootloader）。
*   **OTP (One-Time Programmable)**：一次性可编程存储器。出厂后只能写一次，写完就变 ROM，常用于存放唯一的芯片 ID、安全密钥等。

### 5. 非易失性存储器 (Non-Volatile Memory) —— 长期保存
掉电后数据依然存在，相当于芯片的“硬盘”。
*   **Flash (闪存)**：
    *   **用途**：存放用户的应用程序代码（Firmware）。
    *   **特点**：容量较大，可以多次擦写。现在的蓝牙芯片通常内置 **eFlash** (嵌入式闪存)。
*   **EEPROM (电可擦除可编程只读存储器)**：
    *   **用途**：存放一些需要频繁修改的小型配置数据（如蓝牙配对信息、用户设置）。
    *   **特点**：可以按字节擦写，寿命比 Flash 长，但容量很小。

### 6. 专用缓冲区 (Special Buffers) —— 搬运工
*   **FIFO (First-In-First-Out)**：先进先出缓冲器。常用于通信接口（如 UART, SPI, USB），用来平衡 CPU 处理速度和外部设备传输速度不匹配的问题。
*   **DMA Buffer**：直接存储器访问缓冲区，允许外设直接与内存交换数据，不需要 CPU 全程参与。

---

### 总结对比表

| 存储类型 | 速度 | 容量 | 掉电数据保存 | 主要用途 |
| :--- | :--- | :--- | :--- | :--- |
| **寄存器** | 极快 | 极小 (Byte) | 丢失 | CPU 核心即时运算 |
| **SRAM** | 很快 | 中小 (KB/MB) | 丢失 | 运行中的变量、堆栈 |
| **ROM/OTP** | 中等 | 小 | **保存** | 出厂固件、序列号、密钥 |
| **Flash** | 较慢 | 大 (MB) | **保存** | 存放程序代码 |
| **EEPROM** | 慢 | 很小 (KB) | **保存** | 存放配置参数 |

**形象理解：**
*   **寄存器**：你手里正拿着的笔和纸。
*   **SRAM (RAM)**：你面前宽敞的办公桌，放着正要处理的文件。
*   **Flash/ROM**：办公室的书柜，存放着参考书和档案，用的时候拿出来，不用时一直存着。
