## 仓库结构与关联

```
obsidian_notes/
├── .git/              # Gitee 私人仓库（主仓库）
├── .gitmodules        # submodule 配置
├── 技术笔记/          # GitHub 公开仓库（submodule）
│   ├── .git/
│   └── *.md           # 技术类笔记
├── 日记/              # 私人笔记
└── *.md               # 其他私人笔记
```

| 目录     | 远程仓库                                           | 可见性 |
|----------|----------------------------------------------------|--------|
| 根目录   | `git@gitee.com:wusj1001/obsidian_notes.git`        | 私人   |
| 技术笔记/ | `git@github.com:WuSJ1001/embedded-notes.git`       | 公开   |

## 日常工作流程

- **更新技术笔记并推送到 GitHub**
  ```bash
  cd 技术笔记
  git add .
  git commit -m "更新技术笔记"
  git push origin main
  ```

- **更新私人笔记并推送到 Gitee**
  ```bash
  cd ..
  git add .
  git commit -m "更新私人笔记"
  git push origin main
  ```

- **更新技术笔记 submodule 引用**
  ```bash
  git add 技术笔记
  git commit -m "更新技术笔记 submodule 引用"
  git push origin main
  ```

## Obsidian 使用建议

1. 打开 Obsidian，将整个 `obsidian_notes` 目录添加为库。
2. 日常编辑时按“技术笔记/”“私人笔记”分流提交，避免混合推送。
3. 若多人协作，拉取更新后执行 `git submodule update --init --recursive` 确保子模块同步。

## 双仓协作注意事项

1. **技术笔记** 的文件改动只会推送到 GitHub；Gitee 仅跟踪其 submodule 引用。
2. 拉取远端更新时需分别拉取主仓库与子模块：
   ```bash
   git pull origin main
   git submodule update --init --recursive
   ```
3. 若需备份技术笔记内容到本地，可在主仓库外另建备份仓库，避免直接修改 submodule 路径。

## 子模块解除操作指南

适用场景：`obsidian_notes` 主仓库不再使用 `技术笔记/` 作为 submodule，需将其转为普通目录或移除。

### 操作步骤

1. **确认子模块信息**
   ```bash
   git submodule status
   cat .gitmodules
   ```

2. **视需求清空子目录内容**
   ```bash
   cd <子目录名>
   rm -rf * .*
   cd ..
   ```
   > 若需保留文件，请跳过此步骤并在后续自行备份。

3. **取消子模块注册**
   ```bash
   git submodule deinit -f <子目录名>
   ```

4. **删除子模块的 git 元数据**
   ```bash
   rm -rf .git/modules/<子目录名>
   ```

5. **从 Git 索引移除子模块（关键步骤）**
   ```bash
   git rm --cached <子目录名>
   ```
   > submodule 在索引中以 `160000`（gitlink）存储，必须使用 `--cached` 才能移除引用。

6. **更新 `.gitmodules`**
   ```bash
   rm .gitmodules
   # 或手动删除对应条目
   ```

7. **提交更改并可选推送**
   ```bash
   git add .gitmodules <子目录名>/
   git commit -m "移除嵌套的子模块"
   git push
   ```

### 快速批处理示例（子目录名为 `2`）

```bash
cd 2 && rm -rf * .* && cd ..
git submodule deinit -f 2
rm -rf .git/modules/2
git rm --cached 2
rm .gitmodules
git commit -m "移除嵌套的 github 子模块"
git push
```

### 操作后验证

```bash
git submodule status     # 应无输出
cat .gitmodules          # 应提示文件不存在
```

### 注意事项

| 需求                 | 操作建议                                                |
|----------------------|---------------------------------------------------------|
| 不保留子目录文件     | 执行步骤 2 清空目录                                     |
| 保留子目录文件       | 跳过步骤 2 与步骤 5，改用 `git add <子目录名>` 重新纳入 |
| 保留子目录 git 历史  | 进入子目录执行 `git remote remove origin` 使其独立     |
| 彻底删除子目录       | 步骤 2 后执行 `rmdir <子目录名>`                        |
