# 📝 嵌入式与蓝牙技术笔记：从串口启动到协议栈

## 一、 串口启动 (Serial Boot / UART Boot)

### 1. 定义
串口启动是指嵌入式设备在加电时，不运行内部 Flash 中的程序，而是通过 UART 接口从外部（如 PC）接收代码并运行的过程。

### 2. 工作原理
*   **触发机制**：通过硬件引脚（如 STM32 的 BOOT0）电平状态引导 CPU 执行内置的 **Boot ROM**（出厂预置代码）。
*   **通信过程**：芯片初始化串口，等待握手信号，随后接收数据并写入 SRAM 或 Flash。

### 3. 应用场景
*   **生产烧录**：为空片灌入最初的固件。
*   **系统救砖**：当内部固件损坏无法开机时，强行重写固件。
*   **固件升级 (ISP)**：无需仿真器（J-Link等），仅需串口线即可更新程序。

---

## 二、 串口启动与 HCI 协议的关系

**并不是所有串口启动都用 HCI 协议：**
1.  **通用 MCU**（如 STM32）：使用厂商私有协议或 XMODEM/YMODEM，**不使用 HCI**。
2.  **蓝牙芯片**：在加载固件补丁（Patch）或进入引导模式时，常将数据封装在 **HCI 厂商自定义命令**中，以便复用驱动层代码。

---

## 三、 HCI (Host Controller Interface) 深度解析

HCI 是连接蓝牙 **Host（大脑）** 与 **Controller（手脚）** 的标准化接口。

### 1. HCI 传输层 (Transport Layer)
*   **作用**：在物理连线（如 UART/USB）上为数据包加装“信封”。
*   **标识符**（UART 环境下）：
    *   `0x01`：**HCI Command**（主机发给控制器的指令）
    *   `0x02`：**ACL Data**（普通业务数据）
    *   `0x04`：**HCI Event**（控制器返回给主机的事件反馈）

### 2. HCI Command (HCI 命令)
*   **定义**：Host 向 Controller 下达的具体指令（如：开始扫描、建立连接、复位）。
*   **结构**：包含 **Opcode**（操作码）、**Length**（长度）和 **Parameters**（参数）。
*   **逻辑**：通常遵循“命令 -> 事件”的闭环，即发出 Command 后，Controller 会回一个 `Command Complete` 事件。

### 3. 在单芯片 SoC 中的作用
即使 Host 和 Controller 集成在同一颗硅片上，HCI 依然以**逻辑接口**的形式存在：
*   **架构解耦**：方便更换不同的协议栈（如从厂商栈换成开源栈）。
*   **跨核通信**：在多核芯片（App核与射频核）间充当内部通信协议。
*   **RF 认证测试**：通过 DTM（直接测试模式）收发 HCI 命令，进行射频指标检测。
*   **调试窗口**：通过截获 HCI Snoop Log 分析蓝牙连接问题。

## 四、 蓝牙协议栈 (Bluetooth Protocol Stack)

### 1. 什么是协议栈？
协议栈是运行在硬件之上的**软件算法集合**，它将复杂的无线电波转换为标准化的数据接口，确保不同品牌的设备能互联互通。

### 2. 分层架构
*   **Controller (底层)**：负责射频（PHY）和链路管理（LL），处理最底层的发包和重传。
*   **Host (中层)**：
    *   **L2CAP**：数据分片与合并。
    *   **SMP**：负责配对与加密安全。
    *   **ATT/GATT**：定义数据的存储格式和服务结构（如心率服务）。
    *   **GAP**：管理设备的可见性、角色定义（广播者/扫描者）。
*   **Application (应用层)**：用户编写的具体业务逻辑。

### 3. 常见的实现
*   **BlueZ**：Linux 官方协议栈（如树莓派、安卓早期）。
*   **NimBLE**：开源轻量级，常用于 ESP32 和 RTOS 环境。
*   **SoftDevice**：Nordic 半导体提供的稳定闭源协议栈。

## 💡 核心总结 (Analogy)

| 术语 | 形象比喻 | 核心职责 |
| :--- | :--- | :--- |
| **串口启动** | **强插 U 盘启动电脑** | 在系统损坏或空白时注入程序。 |
| **蓝牙协议栈** | **语言与社交礼仪** | 规定了设备如何打招呼、怎么握手、如何安全交流。 |
| **HCI 传输层** | **快递公司的卡车** | 负责在 Host 和 Controller 之间搬运数据包并分类。 |
| **HCI Command** | **上级的书面指令** | 主机命令蓝牙硬件去执行具体动作。 |
