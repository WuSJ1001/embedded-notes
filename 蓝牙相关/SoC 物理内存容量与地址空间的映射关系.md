# 📝 学习笔记：SoC 物理内存容量与地址空间的映射关系
**—— 以 YC1121S (易兆微) 为例**
#物理内存和地址空间


在1121中，只有0-1000和4000-5000有效
### ❓ 疑问背景
在阅读《YC1121S Datasheet》和《YiChip GPIO 外设说明》时发现一个矛盾：
*   **外设说明规定**：`0x0000` ~ `0x7FFF`（大小为 32KB）对应内部存储，`0x8000` 之后的地址对应外设寄存器。
*   **Datasheet 规格**：明确写着芯片只有 **8KB RAM**。
*   **问题**：8KB 的 RAM 怎么会分出 32KB 的寻址空间？

### 💡 核心结论
**“物理内存容量”** 不等于 **“内存地址寻址空间（Memory Map）”**。
芯片在设计寻址空间时，通常会预留出逻辑地址窗口（如划定 32KB 专区），而实际塞入的物理存储器大小只要不超过这个窗口即可，多余的地址作为“保留区（Reserved）”。

---

### 🔍 详细剖析

#### 1. 内部实际存在的物理存储不止 8KB
仔细研读 Datasheet 第 1 页（Key Features）和第 7 页（Block Diagram），芯片内部挂载在数据总线上的存储模块其实包含三个部分：
*   **8 KB Data RAM**（由 4KB X_RAM_ret 和 4KB DataRAM 组成）
*   **8 KB Patch RAM**（带保持功能的补丁 RAM）
*   **8 KB OTP**（一次性可编程存储器）
**👉 物理存储总计：** 8KB + 8KB + 8KB = **24 KB**。
这 24KB 都是需要 CPU 寻址访问的核心存储区。

#### 2. 地址空间（Memory Map）的区域划分规划
在 32-bit RISC 架构的 SoC 设计中，工程师会对整个地址总线进行“区域划分”（类似于城市规划规划了不同功能区）：
*   **内部存储区（`0x0000` ~ `0x7FFF`）**：总容量上限是 **32 KB**。
    *   上述 **24 KB** 的物理存储（RAM + OTP）被刚好安置在这个区间内。
    *   剩下的 **8 KB** 空间（32KB - 24KB）属于**保留地址（Reserved Space）**。代码不应访问这些空“门牌号”。
*   **外设寄存器区（`0x8000` 起步）**：以此为界线，往后的地址统一分配给硬件外设。
    *   例如 GPIO 寄存器 `core_gpio_oe` 的地址为 `0x8070`，完美坐落于外设寻址区。

#### 3. 延伸思考：框图里的 96KB ROM 去哪了？
既然 `0x0000~0x7FFF` 只有 32KB 的寻址空间，Datasheet 框图中那块 **96KB ROM** 怎么放得下？
通常有以下两种架构解释：
1.  **哈佛架构（Harvard Architecture）设计**：CPU 分为“指令总线”和“数据总线”。ROM 存储代码指令，走的是独立的指令地址空间；而 RAM/OTP 和外设走的是数据地址空间。两者门牌号互不干扰。
2.  **映射在高位地址**：在纯冯·诺依曼架构下，96KB ROM 会被映射到外设区上方更高的地址段（例如 `0x10000` 之后）。

### 🌟 总结 / Takeaways
在进行底层嵌入式开发或阅读芯片手册时：
1. **地址范围差值 ≠ 实际物理内存大小**。分配给某类存储的逻辑地址空间，往往大于它实际的物理硅片面积（容量），多出来的是保留寻址空间。
2. 看 Memory 规格时，**要看全面**。不能只看 "Data RAM"，还要注意 OTP、Patch RAM、FIFO 等其他挂在数据总线上的存储外设。
