## 1. 芯片概述

### 1.1 YC1121S 主要特性

| 特性 | 规格 |
|------|------|
| MCU | 24 MHz 32 位专有 MCU |
| 闪存 | 8 KB OTP (一次性可编程) |
| RAM | 8 KB 数据 RAM (4 KB 保持) + 8 KB Patch RAM |
| 无线 | Bluetooth 5.4 / Long Range |
| GPIO | 最多 24 个 GPIO，功能复用 |
| ADC | 9 通道 12 位 SAR ADC |
| 其他 | PWM×8, I2C, SPI, UART, USB 2.0 |

### 1.2 关键引脚

| 引脚 | 名称 | 功能 |
|------|------|------|
| Pin 39 | GPIO21/ADC | 通用 GPIO，支持 ADC |
| Pin 1 | GPIO23 | 通用 GPIO，具有 EXEN 复用功能 |
| Pin 2 | RSTN | 全局复位，低电平有效 |
| Pin 8-9 | XTAL_OUT/IN | 24 MHz 晶振 |

## 3. 哈佛架构与补丁机制

### 3.1 哈佛架构特点

YC1121S 采用**哈佛架构**，程序存储器和数据存储器分开：
- **程序存储器**: OTP + Patch RAM
- **数据存储器**: Data RAM

### 3.2 补丁机制原理

由于 OTP 烧录后不可修改，芯片设计了**补丁 (Patch) 机制**：

1. **补丁点**: 64 个预定义的补丁入口点 (`patch00_0` ~ `patch3f_7`)
2. **补丁标志位**: `mem_patch00` ~ `mem_patch3f`，每个变量的每一位控制一个补丁点
3. **补丁使能**: 设置对应标志位后，程序执行到补丁点时会跳转到补丁函数

### 3.3 补丁点命名规则

```
patchXX_Y
  ├── XX: 00-3f (对应 mem_patch00-mem_patch3f)
  └── Y:  0-7   (对应变量的第 0-7 位)

示例:
  patch05_0 → mem_patch05 的第 0 位
  patch1b_3 → mem_patch1b 的第 3 位
```

### 3.4 补丁检查代码示例

```asm
; patch.prog 中的补丁检查
beq patch00_0, p_soft_reset    ; 如果 mem_patch00 bit0=1，跳转到 p_soft_reset
beq patch05_0, p_led_gpio_patch ; 如果 mem_patch05 bit0=1，跳转到 p_led_gpio_patch
```

## 5. 汇编语言基础

### 5.1 寄存器

| 寄存器 | 位宽 | 说明 |
|--------|------|------|
| `pdata` | 64 位 | 通用寄存器，fetch/ifetch 默认目标 |
| `temp` | 64 位 | 通用寄存器 |
| `rega` | 32 位 | 通用寄存器 |
| `regb` | 32 位 | 通用寄存器 |
| `regc` | 17 位 | 通用寄存器 |
| `queue` | 8 位 | 配合 qset 等指令使用 |
| `loopcnt` | 32 位 | 循环计数器 |
| `mark` | 64 位 | 用户标志寄存器 |

### 5.2 常用指令

#### 5.2.1 数据传送

```asm
; 立即数传送
setarg 5              ; 将 5 放入 pdata
arg 5, temp           ; 将 5 放入 temp
jam 0x01, 0x100       ; 将 0x01 写入内存地址 0x100

; 内存读写
fetch 1, 0x100        ; 从 0x100 读 1 字节到 pdata
fetcht 1, 0x100       ; 从 0x100 读 1 字节到 temp
store 1, 0x100        ; 将 pdata 的 1 字节写入 0x100
storet 1, 0x100       ; 将 temp 的 1 字节写入 0x100

; 寄存器复制
copy rega, pdata      ; 将 pdata 复制到 rega
```

#### 5.2.2 位操作

```asm
set1 5, pdata         ; 将 pdata 第 5 位置 1
set0 5, pdata         ; 将 pdata 第 5 位清 0
setflip 5, pdata      ; 将 pdata 第 5 位取反
isolate1 5, pdata     ; 检查 pdata 第 5 位，为 1 则 true=1
lshift pdata, pdata   ; pdata 左移 1 位
rshift pdata, pdata   ; pdata 右移 1 位
```

#### 5.2.3 运算指令

```asm
iadd rega, regb       ; regb = regb + pdata
isub rega, regb       ; regb = pdata - rega
imul32 temp, pdata    ; pdata = pdata * temp
div pdata, 10         ; pdata / 10
idiv rega             ; pdata / rega
```

#### 5.2.4 跳转指令

```asm
call function_name    ; 调用子程序
rtn                   ; 子程序返回
branch label          ; 无条件跳转
beq 5, label          ; pdata==5 则跳转
bbit1 3, label        ; pdata 第 3 位为 1 则跳转
bbit0 3, label        ; pdata 第 3 位为 0 则跳转
rtnne 5               ; pdata!=5 则返回
loop label            ; loopcnt-1，不为 0 则跳转
```

#### 5.2.5 标志位

| 标志位 | 说明 |
|--------|------|
| `zero` | 运算结果为 0 时置 1 |
| `positive` | 运算结果大于 0 时置 1 |
| `blank` | pdata 为 0 或无数据时置 1 |
| `true` | 逻辑运算结果为真时置 1 |
| `wake`, `match`, `sync` | 用户/硬件标志位 |

### 5.3 条件语句结构

```asm
; if 结构
fetch 1, mem_flag
rtn zero              ; 如果为 0 则返回（不执行）
; ... 条件满足时执行的代码

; if-else 结构
fetch 1, mem_flag
bbit1 0, if_true      ; 第 0 位为 1 则跳转
; ... else 分支
branch if_end
if_true:
; ... if 分支
if_end:

; switch 结构 (多分支)
fetch 1, mem_value
beq 1, case1
beq 2, case2
beq 3, case3
rtn

case1:
; ...
rtn
case2:
; ...
rtn
```

### 5.4 循环结构

```asm
; for 循环
arg 10, loopcnt       ; 循环 10 次
loop_start:
; ... 循环体
loop loop_start       ; loopcnt-1，不为 0 则跳转

; while 循环
while_loop:
fetch 1, mem_cond
rtn zero              ; 条件为 0 则退出
; ... 循环体
branch while_loop
```

## 6. GPIO 配置与使用

### 6.1 GPIO 相关寄存器

| 寄存器 | 地址 | 说明 |
|--------|------|------|
| `core_gpio_conf` | 0x8070 | GPIO 方向配置 (低 16 位) |
| `core_gpio_conf_hi` | 0x80d4 | GPIO 方向配置 (高 8 位) |
| `core_gpio_out` | - | GPIO 输出数据 |
| `core_gpio_in` | 0x811c | GPIO 输入数据 (低 16 位) |
| `core_gpio_in1` | 0x811d | GPIO 输入数据 (高 8 位) |
| `core_gpio_ds` | 0x8049 | GPIO 驱动强度 |

### 6.2 GPIO 配置类型

```asm
; 定义于 regs.format
gpcfg_input         = 0     ; 输入
gpcfg_output_low    = 62    ; 输出低电平
gpcfg_output_high   = 63    ; 输出高电平
gpcfg_pullup        = 0x40  ; 上拉
gpcfg_pulldown      = 0x80  ; 下拉
gpcfg_high_impedance = 0x00 ; 高阻态

; 复用功能
gpcfg_uart_txd      = 8
gpcfg_uart_rxd      = 9
gpcfg_pwm_out0      = 16
gpcfg_spid_mosi     = 34
; ... 等等
```

### 6.3 GPIO 配置函数

`peripherals.prog` 中提供了标准配置函数：

```asm
; 配置 GPIO 函数 (通用)
; pdata = GPIO 编号 | 配置类型
setarg 21
or_into gpcfg_output_low, pdata
call gpio_config_function

; 简化配置
setarg 21
call gpio_config_output_low   ; 配置为输出低电平

setarg 23
call gpio_config_input_pu     ; 配置为输入带上拉
```

### 6.4 GPIO 读写

```asm
; 读取 GPIO 输入
fetch 1, core_gpio_in
isolate1 7, pdata       ; 检查第 7 位 (GPIO23)
rtnne true              ; 如果为高电平则返回

; 设置 GPIO 输出
set1 21, core_gpio_out  ; GPIO21 输出高电平
set0 21, core_gpio_out  ; GPIO21 输出低电平
```

### 6.5 硬件连接示例

```
LED 连接 (GPIO21):
  GPIO21 --- [220Ω电阻] --- LED 阳极 --- LED 阴极 --- GND

按键连接 (GPIO23):
  GPIO23 --- 按键 --- GND
  (内部上拉，按下时为低电平)
```

## 7. 补丁点使用完整流程

### 7.1 选择补丁点

1. 查看 `patch.prog` 中已被使用的补丁点
2. 选择空闲的补丁点（如 `patch05_0`）

```asm
; 已使用的补丁点示例
beq patch00_0, p_soft_reset      ; bit0
beq patch02_1, p_set_sync_on     ; bit1
beq patch05_0, p_led_gpio_patch  ; 空闲，可使用
```

### 7.2 步骤 1：定义内存变量

在 `format/bt.format` 的 `xmemalloc` 段添加：

```asm
xmemalloc(
// 其他变量...

// LED GPIO 控制程序变量
1 mem_led_gpio_init_flag
1 mem_led_status
)
```

### 7.3 步骤 2：编写补丁函数

创建 `program/my_feature/led.prog`：

```asm
; ================================================================
; LED GPIO 补丁函数
; ================================================================
p_led_gpio_patch:
    ; 检查是否已初始化
    fetch 1, mem_led_gpio_init_flag
    beq 0x01, p_led_gpio_check  ; 已初始化则跳转

    ; 首次调用时初始化
    set1 0, mem_led_gpio_init_flag

    ; GPIO 配置...
    setarg 21
    call gpio_config_output_low
    setarg 23
    call gpio_config_input_pu
    jam 0, mem_led_status
    rtn

p_led_gpio_check:
    ; 开关检测逻辑...
    fetch 1, core_gpio_in
    isolate1 7, pdata
    rtnne true
    call p_toggle_led
    call p_wait_switch_release
    rtn
```

### 7.4 步骤 3：在 patch.prog 中添加调用

```asm
; patch.prog 补丁表
beq patch05_0, p_led_gpio_patch  ; 添加补丁点调用

; 文件末尾包含源文件
INCLUDE "my_feature/led.prog"
```

### 7.5 步骤 4：启用补丁点

在初始化代码中设置补丁标志位：

```asm
; 在 app_init 或合适的位置添加
set1 0, mem_patch05    ; 启用 patch05_0 补丁点
```

或者在 `bt.prog` 的 `main_loop` 之前添加。

### 7.6 补丁点使能位总览

| 补丁点 | 标志位变量 | 位 |
|--------|-----------|-----|
| patch00_0 | mem_patch00 | bit 0 |
| patch05_0 | mem_patch05 | bit 0 |
| patch1b_3 | mem_patch1b | bit 3 |
| ... | ... | ... |

## 8. 实战示例：开关控制 LED

### 8.1 功能需求

- GPIO21 连接 LED（输出）
- GPIO23 连接按键（输入）
- 检测到按键按下后翻转 LED 状态

### 8.2 完整代码

```asm
// ================================================================
// YC1121S GPIO 演示程序
// 功能：检测 GPIO23 开关按下，翻转 GPIO21 LED 状态
// ================================================================

p_led_gpio_patch:
    // 检查是否已初始化
    fetch 1, mem_led_gpio_init_flag
    beq 0x01, p_led_gpio_check

    // 首次调用时初始化 GPIO
    set1 0, mem_led_gpio_init_flag

    // 配置 GPIO21 为输出模式，初始为低电平（LED 灭）
    setarg 21
    call gpio_config_output_low

    // 配置 GPIO23 为输入模式，带上拉电阻
    setarg 23
    call gpio_config_input_pu

    // 初始化 LED 状态变量
    jam 0, mem_led_status
    rtn

// ================================================================
// 开关检测函数
// ================================================================
p_led_gpio_check:
    fetch 1, core_gpio_in        // 读取 GPIO 输入
    isolate1 7, pdata            // 检查第 7 位 (GPIO23)
    rtnne true                   // 高电平 (未按下) 则返回

    call p_toggle_led            // 翻转 LED
    call p_wait_switch_release   // 等待释放
    rtn

// ================================================================
// 翻转 LED 函数
// ================================================================
p_toggle_led:
    fetch 1, mem_led_status
    setflip 0, pdata
    store 1, mem_led_status
    beq 0, p_led_off

p_led_on:
    set1 21, core_gpio_out
    rtn

p_led_off:
    set0 21, core_gpio_out
    rtn

// ================================================================
// 等待开关释放 (带防抖)
// ================================================================
p_wait_switch_release:
    nop 5000                     // 消抖延时 (~417us)
    fetch 1, core_gpio_in
    isolate1 7, pdata
    rtnne true

p_wait_release_loop:
    nop 1000
    fetch 1, core_gpio_in
    isolate1 7, pdata
    rtnne true
    branch p_wait_release_loop
    nop 3000
    rtn

// ================================================================
// GPIO 配置辅助函数
// ================================================================
gpio_config_output_low:
    copy pdata, temp
    and temp, 0x1f, temp
    call gpio_addr
    setarg gpcfg_output_low
    istore 1, contw
    rtn

gpio_config_input_pu:
    copy pdata, temp
    and temp, 0x1f, temp
    call gpio_addr
    setarg gpcfg_input|gpcfg_pullup
    istore 1, contw
    rtn
```

## 10. 常见问题与注意事项

### 10.1 补丁不工作

**问题**: 补丁函数没有被调用

**检查清单**:
1. 补丁标志位是否正确设置 (`set1 Y, mem_patchXX`)
2. `patch.prog` 中是否添加了对应的 `beq` 语句
3. 是否通过 `INCLUDE` 包含了源代码文件
4. 函数标签名称是否与 `beq` 中的一致

### 10.2 GPIO 配置无效

**问题**: GPIO 输出/输入不正常

**检查清单**:
1. 确认 GPIO 编号正确 (0-23)
2. 检查是否配置了正确的功能模式
3. 确认没有与其他外设功能冲突 (复用)
4. 检查硬件连接是否正确

### 10.3 内存变量访问错误

**问题**: 程序运行异常或崩溃

**检查清单**:
1. 变量是否在 format 文件中正确定义
2. 变量大小是否匹配 (1/2/4 字节)
3. 避免使用已定义的变量名
4. 检查变量地址是否与其他变量重叠

### 10.4 延时时间计算

```asm
nop 12    ; 12 * (1/12)us = 1us  (24MHz 晶振)
nop 120   ; 10us
nop 6000  ; 500us
nop 12000 ; 1ms
```

### 10.5 编程规范建议

1. **注释**: 每个函数添加功能说明
2. **命名**: 函数名使用小写 + 下划线，变量使用 `mem_` 前缀
3. **模块化**: 将功能代码放在独立文件中
4. **备份**: 修改前备份原始文件
5. **测试**: 每次修改后充分测试

### 10.6 调试技巧

1. 使用 `debug` 寄存器输出中间值
2. 通过 LED 闪烁显示程序执行状态
3. 使用 ICE/SWD 调试器单步执行
4. 利用 UART 打印调试信息
