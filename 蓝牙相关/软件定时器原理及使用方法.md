# YC1121S 软件定时器原理及其使用方法

---

## 一、时间寄存器说明

YC1121S 汇编芯片**不支持定时器中断**，所有软件定时均通过**查询方式**实现。

---

### 1.1 clkn（时间寄存器，只读）

| 项目 | 说明 |
|------|------|
| **位宽** | 16 bits（clkn_rt）+ 28 bits（clkn_bt） |
| **特性** | 随芯片运行不断自增，上电时从 0 开始计数 |
| **驱动源** | 晶振或 DPLL |
| **备注** | 蓝牙连接时，Master 的时钟；Slave 以该时钟为校正时钟 |

```
clkn 整体结构：
┌──────────────────┬──────────────────┐
│   clkn_bt (28b)  │   clkn_rt (16b)  │
└──────────────────┴──────────────────┘
```

---

### 1.2 clkn_bt（标准蓝牙时钟，只读）

| 项目 | 说明 |
|------|------|
| **位宽** | 28 bits |
| **特性** | 随芯片运行不断自增，达到最大值后从 0 重新开始 |
| **本质** | clkn 寄存器的高 28 位 |
| **精度** | 312.5 µs（对应频率 3.2 kHz） |

---

### 1.3 clkn_rt（系统时钟，只读）

| 项目 | 说明 |
|------|------|
| **位宽** | 16 bits |
| **特性** | 随芯片运行不断自增，达到最大值后向 clkn_bt 进位 |
| **本质** | clkn 寄存器的低 16 位 |
| **精度** | 1 / 系统时钟（如系统时钟为 12 MHz，则精度为 1/12 µs） |
| **取值范围** | 0 ~ 3749，超出则向 clkn_bt 进位 |

---

### 1.4 clkn_bt 与 clkn_rt 的换算关系

**换算公式：**
```
Time = (clkn_bt × 3750 + clkn_rt) × (1 / 系统时钟)
```

**示例：** clkn 从 0 运行 1000000 个 clock（1/12 µs）= 83333 µs

| 寄存器 | DEC 值 | HEX 值 |
|--------|--------|--------|
| clkn_bt | 266 | 0x10a |
| clkn_rt | 2500 | 0x9c4 |
| clkn（整体） | — | 0x000010a09c4 |

```
Time = (266 × 3750 + 2500) × (1/12) µs = 83333 µs
```

---

### 1.5 clke（可读可写寄存器）

clke 与 clkn 结构相同，同样分为 clke_bt（28 bits）和 clke_rt（16 bits）：

| 寄存器     | 说明                                             |
| ------- | ---------------------------------------------- |
| clke_bt | 标准蓝牙时钟，可读可写                                    |
| clke_rt | 精度同 clkn_rt（系统时钟），取值范围 0 ~ 3749，超出向 clke_bt 进位 |

---

### 1.6 为什么 clkn_bt 运行 3.2 kHz，clkn_rt 自由运行 12 MHz？

当系统时钟为 12 MHz 时，clkn_rt 的精度为 **1/12 µs**：

```
3750 × (1/12 µs) = 312.5 µs
```

312.5 µs 正好是 clkn_bt 的精度，也正是蓝牙需要的基础时钟：

```
F = 1 / 0.0003125 s = 3200 Hz = 3.2 kHz
```

因此，ROM 现成的软件定时器标号基于 **312.5 µs × 2 = 0.625 ms** 实现；100 ms 软件定时器则基于 **312.5 µs** 的 clkn_bt 通过查询方式实现。

---

## 二、0.625 ms 定时器

### 2.1 定时器标号

YC1121S 的 0.625 ms 定时器基于 BT_CLK，共有 **5 个定时器标号**，可直接使用的为：**0、2、9、7、11**。
```
0 le_adv_interval_timer
// 广播发送定时器
2 le_scan_interval_timer
// 广播扫描定时器，需开启扫描模式
9 key_scan_timer
// 扫键定时器
7 smp_pairing_timer
// 配对定时器
11  ui_led_blink
// 闪灯定时器
```

### 2.2 使用举例：定时发送广播

**操作步骤：**

| 步骤  | 操作               | 说明                                        |
| --- | ---------------- | ----------------------------------------- |
| 1   | 调用 `timer_init`  | 记录当前时间，将目标定时时间写入 `pdata`，将定时器标号写入 `queue` |
| 2   | 调用 `timer_check` | 在需要检查的地方将定时器标号写入 `queue`，调用检查函数           |
| 3   | 判断               | 定时时间到达后，`blank` 标志位置 1，软件据此执行相应动作         |

## 三、100 ms 软件定时器

### 3.1 实现原理（基于 clkn_bt 查询）

所有软件定时通过**查询 clkn_bt** 实现，步骤如下：

```
1. 记录初始时刻 t1
   └─ 在需要开始计时的位置，读取 clkn_bt 当前值 → t1

2. 获取当前时刻 t2
   └─ 在需要检测定时的位置，读取 clkn_bt 当前值 → t2

3. 计算时间差 delta_t
   ├─ 若 t2 >= t1：delta_t = t2 - t1
   └─ 若 t2 < t1：delta_t = 0x10000000 - t1 + t2（回环处理）

4. 判断定时是否到达
   ├─ delta_t > t_set → 定时到达，执行任务
   └─ 否则 → 返回步骤 2，刷新 t2 重新计算 delta_t
```

### 3.2 代码实现

#### 3.2.1 定时检测函数 `ui_timer_check`

通过查询 clkn_bt 的方式判断定时时间是否已满足。

#### 3.2.2 定时到达回调 `app_event_timer`

定时时间到达后，通过回调函数执行对应任务。

#### 3.2.3 回调函数实现步骤

| 步骤 | 说明 |
|------|------|
| 1 | 初始化时，将定时任务函数的入口地址存入变量 `mem_cb_event_timer` |
| 2 | 在 `mainloop` 中通过 `ui_timer_check` 查询 clkn_bt，判断是否满足定时 |
| 3 | 定时满足时，从 `mem_cb_event_timer` 取出函数入口地址，将 PC 指针跳转过去，实现回调 |

## 四、核心要点速记

| 要点 | 内容 |
|------|------|
| **定时方式** | 查询（非中断） |
| **时间基准** | clkn_bt = 312.5 µs（3.2 kHz） |
| **100 ms 定时** | 基于 clkn_bt 查询，t_set = 100 ms / 312.5 µs ≈ 320 |
| **0.625 ms 定时** | 基于 BT_CLK，可用法 0/2/9/7/11 五个标号 |
| **clkn 组成** | clkn_bt（高28位）+ clkn_rt（低16位，精度 1/系统时钟） |
| **回环处理** | 当 t2 < t1 时，delta_t = 0x10000000 - t1 + t2 |
